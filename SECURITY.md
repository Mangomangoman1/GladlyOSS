# Security Policy

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting flow:

<https://github.com/Mangomangoman1/GladlyOSS/security/advisories/new>

Include a clear description, reproduction steps, and the potential impact when possible. Maintainers will acknowledge your report as soon as they can and keep you informed while it is reviewed.

## Supported versions

Gladly is currently in early development. Security fixes are applied to the latest version on the default branch.
